Game Demo - Lofi City

Lofi City is an ongoing solo project that showcases my current skills in game development and the Unity Engine. I am building a farming, automation, cooking and shop management game. This section shows the progress made over various months.

The main focus is on creating the base mechanics for the game. The mechanics that have been created are:

  • A multi-layered grid and building system, which allows for adding height to the world layout.
  • An item and inventory system. This makes use of scriptable objects and addressable assets.
  • An automation system that works by connecting machines and queuing crafting requests.
  • Farming and field cleaning systems, where the player can cultivate different ecosystems. These were designed around Solarpunk.
  • Street vending, which lets players sell items to NPCs, with the ability to adjust the price based on demand.
  • A cooking system, where players need to prepare food while serving customers.
  • A date and time system, with a reactive world.
  • A quest system, which detects when players complete a task.
  • 2D character rigs for background NPCs.
  • NPC pathfinding, schedule and personality system. This determines where they go and what food they want.
  • A JSON based save system, with different save slots.
  • The Yarn Spinner dialogue system has been integrated into the project.
  • Various smaller features, such as screen transitions, integrating audio and in-world shops
  • Shaders and Custom Lighting (Work in Progress)

A short browser-based demo is available to play on Itch.io. Alternatively, a gameplay video of the demo is also available on YouTube. This demo is somewhat outdated, but still showcases a number of game mechanics.

I created a game design document to keep track of the different mechanics. The MDA (Mechanics, Dynamics, Aesthetic) framework was used when designing the game mechanics, so that they better align with the intended themes. The GDD as well as the technical documentation can be found here.

Emphasis was placed on maintaining a clean and documented code base, as well as considering the performance of the system. Focus was placed on the structure and decoupling of the code. This made it significantly easier to iteratively alter the code later on. GitHub was used for source control, which also allowed for prototyping mechanics on separate branches. A Trello board was used throughout development to keep track of tasks.



Thesis Project

My undergraduate thesis was focused on neural style transfer. As part of my project, I explored efficient ways for games to change their art style in real time. I used Unity to create a proof of concept for both 2D and 3D scenarios.

It should be noted that the AI model used was trained on artworks from the public domain. I do not intend to disrespect the rights and efforts of artists.

Existing methods stylise each frame in real time, which requires powerful hardware. My system instead stylises the individual elements, avoiding repetition. For 2D, this was done by loading and swapping stylised sprites. This was extended for 3D by stylising the textures applied to a material. Implementing this system in Unity required a good understanding of how sprites and textures work to allow for changing them efficiently.

The intended use of this system is to easily apply filters to game scenes. The final result achieved the desired goals for both 2D and 3D scenarios. The project was well received, receiving an “A” grade from the university and being nominated for the IEEE Final-Year Project in ICT.

Game Jams

Wowie Jam 4.0

To challenge my skills, I took part in the "Wowie Jam 4.0" hosted by Jonas Tyroller. This was a three day game jam, with the given theme being “Collaborating with AI”. My submission for this was a story-focused shop management game. All aspects of this project (except for the audio creation) were made by me within the given time frame.

Before submission, a significant issue with the dialogue system was found that would have made the game unplayable when built. I was able to rework the entire system in a few hours, changing all the conversations from scriptable objects to JSON files. This showcases my ability to work in a limited time period and being able to handle unexpected issues.

Despite being my first game jam, the submission was relatively well received, being ranked 191st from 1019 entries.

Flight Sim Dashboard

This project was the result of a University assignment. My team and I were given the task to build a dashboard for flight simulation enthusiasts to use alongside the game, displaying helpful information when it is needed. This task aimed to test us on the entire software development process, including the UI/UX design, requirements elicitation, testing and documentation.

The project was built as a web based application using the ASP .Net Core Framework. A scrum board on Trello was used to manage the development progress, and GitHub was used to maintain the codebase.

  • My Main Contributions
  • Coordinated the Project and Managed the Trello Board
  • Research of the Flight Phases From Climb to Approach
  • Setup of the Base Application and Page Layout
  • Connected the Application to Various Third Party APIs
  • Connected a Database and Created an Account System
  • Created Many Different Components for the System

Godot Wild Jam #68

In early 2024, I started to practice game development using the Godot Engine. In April, I took part in the Godot Wild Jam to push myself further in this regard. The jam was nine days long, which gave me enough time to experiment with features of the engine that I had not previously used.
