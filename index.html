Thesis Project

My undergraduate thesis was focused on neural style transfer. As part of my project, I explored efficient ways for games to change their art style in real time. I used Unity to create a proof of concept for both 2D and 3D scenarios.

It should be noted that the AI model used was trained on artworks from the public domain. I do not intend to disrespect the rights and efforts of artists.

Existing methods stylise each frame in real time, which requires powerful hardware. My system instead stylises the individual elements, avoiding repetition. For 2D, this was done by loading and swapping stylised sprites. This was extended for 3D by stylising the textures applied to a material. Implementing this system in Unity required a good understanding of how sprites and textures work to allow for changing them efficiently.

The intended use of this system is to easily apply filters to game scenes. The final result achieved the desired goals for both 2D and 3D scenarios. The project was well received, receiving an “A” grade from the university and being nominated for the IEEE Final-Year Project in ICT.

C# Engine Programming

For this University assignment, a group of four students was given a very simplistic C# game engine and instructed to improve it in any way. We decided to create an engine that could handle more complicated features in 2D games, including a particle system and tilemaps.

To test the engine, we successfully recreated a 2D version of the main mechanic in the game Viewfinder. Players can take a photo of any part of the level, and recreate it anywhere else. This system works with both tilemaps and game objects, and accounts for the parallax effect on sprites.

  • Implemented tile maps and integrated Tiled
  • Assisted with the animation controller and scene management
  • Implemented the Viewfinder-style gameplay mechanic
  • Created most of the art for the demo


Global Game Jam (Unity)

About the game

As I had not worked on an action game previously, I took the opportunity to implement a bullet hell game, with a combat system inspired by Nine Sols. Players can dash through enemies to freeze them, then trigger damage to all frozen enemies. Since more damage is dealt with a longer chain of enemies, players would want to keep multiple enemies on screen, which makes dodging bullets more difficult. In addition, I have added a story to the game which is told in between waves.

Development Process

This was developed solo in Unity. Most of the time was spent on making the movement system fun to use, and on the enemy system. I am quite happy with how these turned out, especially since the enemy movement/attack patterns and wave system were made to be scalable and easy to design using scriptable objects.

Given more time, I would have liked to improve the visuals and vfx of the game, particularly in adding more feedback when triggering an attack.

Board Game Design - University Project

About the project

This was a university assignment, where we needed to design, prototype and playtest a board game. We were inspired by how "Overcooked" requires all players to communicate clearly under difficult situations. To try and recreate this, we made a two player game where each player must work in sync with each other to complete a recipe in time. However, players will occasionally aquire temporary "trait cards", which restrict their communication. For instance, players may only say five words during a turn, or can only draw what they want to say.

My contributions

As all the members were involved in the design and playtesting of the game, most aspects were a team effort. I designed most of the trait cards, which proved to be a very enjoyable part of the game. In addition, I designed the components for players to keep track of their turn, which helped to make a complicated turn order quite easy to follow.
