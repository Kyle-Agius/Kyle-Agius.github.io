Game Demo - Lofi City

This is a solo project that showcases my current abilities. I am building a small slice of a factory building, farming and item shop game. This was done using the Unity Game Engine. Most of the code, documentation and artwork was made from scratch in the span of 6-7 months.

During the first 4 months, I created most of the base mechanics for the game. This included:

  • A Multi-layered Grid System for Building and Player Movement
  • Item and Inventory System Using Scriptable Objects and Addressable Assets
  • Factory System with Connected Machines and Crafting Requests
  • Field Cleaning System with Varying Ecosystems
  • Sell Items to NPCs Through Street Vending
  • Date and Time System
  • NPC Pathfinding, Schedule and Personality System
  • JSON Based Save System with Save Slots
  • Auto Generating Code Documentation Using docfx

An estimated 5 weeks were then spent on creating a short, playable demo for the game. This required adding visuals to the game, as well as a general structure to guide the player. The following tasks were focused on:

  • Creating and Adding the 2D Art to the Project (Not AI Generated)
  • Added 2D Character Rigs for NPCs and Frame-by-Frame Animation for the Player
  • A Quest System to Guide the Player, and Introduce the Different Mechanics
  • Yarnspinner was Integrated for a Tutorial System, and for the Opening Credits
  • Royalty Free Music and Sound Effects were Integrated
  • Various Smaller Features, Such as Screen Transitions and In-World Shops

After completing the first demo, I decided to focus on creating a proper game design document. As such, I spent three weeks adding this to my existing documentation system. The MDA (Mechanics, Dynamics, Aesthetic) framework was used throughout the documentation. This process allowed me to redesign the main systems so that they better align with the intended themes of the game.

Following this, I identified the areas of the game that needed to be more engaging. The first change was related to the cleaning system, which was redisigned to be closer to the intended Solarpunk theme. This and the farming system are now more focused on problem solving.

When playtesting, I fould that the selling system was not particularly engaging, despite the underlying complexity in the price and demand system. As such, the item selling system was changed to be more focused on selling food. A more complex cooking system was added to the street vending system. This should give the player more to manage when street vending, as they need to prepare food while serving customers.

Emphasis was placed on maintaining a clean and documented code base, as well as considering the performance of the system. Focus was placed on the structure and decoupling of the code. This made it significantly easier to iterate and alter the code later on. GitHub was used for source control, which also allowed for prototyping mechanics on separate branches.

Thesis Project

My undergraduate thesis was focused on neural style transfer. As part of my project, I explored efficient ways for games to change their art style in real time. I used Unity to create a proof of concept for both 2D and 3D scenarios.

Existing methods stylise each frame in real time, which requires powerful hardware. My system instead stylises the individual elements, avoiding repetition. For 2D, this was done by loading and swapping stylised sprites. This was extended for 3D by stylising the textures applied to a material. The final result achieved the desired goals for both scenarios.

The project was well received, receiving an “A” grade from the university and being nominated for the IEEE Final-Year Project in ICT.

3 Day Game Jam

To improve my skills in completing a game, I took part in the "Wowie Jam 4.0" hosted by Jonas Tyroller. This was a three day game jam, with the given theme being “Collaborating with AI”. My submission for this was a story-focused shop management game. Despite being my first game jam, the submission was relatively well received, being ranked 191st from 1019 entrees.

Before submission, a significant issue with the dialogue system was found that would have made the game unplayable when built. I was able to rework the entire system in a few hours, changing all the conversations from scriptable objects to JSON files. This showcases my ability to work in a limited time period and being able to handle unexpected issues.

All aspects of this project, except for the music and sound effects, were made by me.

  • Main Features Built
  • Resource Management and Shop Management System.
  • Dialogue Based Story System

Flight Sim Dashboard

This project was the result of a University assignment. My team and I were given the task to build a dashboard for flight simulation enthusiasts to use alongside the game, displaying helpful information when it is needed. This task aimed to test us on the entire software development process, including the UI/UX design, requirements elicitation, testing and documentation.

The project was built as a web based application using the ASP .Net Core Framework. A scrum board on Trello was used to manage the development progress, and GitHub was used to maintain the codebase.

  • My Main Contributions
  • Coordinated the Project and Managed the Trello Board
  • Research of the Flight Phases From CLimb to Approach
  • Setup of the Base Application and Page Layout
  • Connected the Application to Various Third Party APIs
  • Connected a Database and Created an Account System
  • Created Many Different Components for the System
